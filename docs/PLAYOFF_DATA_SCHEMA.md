# Playoff Data Schema

This branch adds a manual-only playoff analysis layer without changing the live display path.

## Guardrails

- `index.html`, `data/yodeck_display.json`, `data/headlines.json`, `aahl_yodeck_processor.py`, and the GitHub Actions workflow remain unchanged in this pass.
- Existing generated files keep their current shape:
  - `data/results.json`
  - `data/schedule.json`
  - `data/player_registry.json`
  - `data/weekly_report.json`
- New playoff and analysis metadata lives only under `data/v2/`.

## Canonical Game Rules

- Halifax local time (`America/Halifax`) is the authoritative league date.
- Canonical game identity is derived from:
  - the two team slugs
  - the Halifax-local start timestamp
  - the location
- Duplicate scoreboard rows with different `game_id` values collapse into a single canonical game.
- When duplicates exist, the preferred canonical record keeps:
  - populated `start_utc`
  - the stable ICS-backed `box_score_url` / `summary_url`
  - the richer scoreboard or player-stat payload

## Legacy File Semantics

- `data/results.json` and `data/schedule.json` stay shape-stable for current consumers.
- Their content is corrected, not expanded:
  - duplicate finals are removed
  - local dates remain Halifax-correct even when UTC rolls into the next day
- `scripts/build_player_registry.py` and `scripts/generate_weekly_report.py` dedupe on canonical game identity before aggregating stats or form.

## Playoff Config

Tracked file: `src/aahlscraper/data/playoffs.json`

Current contract:
- `season_id`
- `timezone`
- `rounds[]`
- Per round:
  - `round_id`
  - `label`
  - `best_of`
  - `start_date`
  - `series[]`
- Per series:
  - `series_id`
  - `higher_seed`, `higher_seed_slug`, `higher_seed_name`
  - `lower_seed`, `lower_seed_slug`, `lower_seed_name`
  - `known_game_dates[]`
  - `notes`

The schedule feed does not expose playoff metadata directly, so playoff phase is assigned from this config plus canonical matchup/date matching.

## `data/v2/` Outputs

Generated by: `python3 scripts/generate_season_analysis.py`

Files:
- `data/v2/games.json`
  - compact canonical game rows
  - includes `canonical_game_id`, `source_game_ids`, local/UTC timestamps, phase/series assignment, scores, and notable performers
- `data/v2/series.json`
  - best-of-3 series state
  - includes completed games, upcoming games, win counts, next game, elimination pressure, and regular-season head-to-head context
- `data/v2/season_overview.json`
  - league-wide standings snapshot, compact leaderboards, head-to-head matrix, and playoff round summary
- `data/v2/teams/<slug>.json`
  - per-team dossier with standing, leaders, recent results, upcoming games, head-to-head summary, and playoff context
- `data/v2/season_analysis.md`
  - deterministic markdown briefing built from the same structured layer

## Manual Workflow

Recommended manual run order on this branch:

```bash
python3 scripts/aahl_cli.py scrape --backend http --targets schedule results stats standings rosters teams
python3 scripts/build_player_registry.py
python3 scripts/generate_season_analysis.py
```

`generate_weekly_report.py` is hardened for dedupe and Halifax-local date handling, but it is not required for the `data/v2/` analysis layer.
